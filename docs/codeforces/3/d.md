# [Least Cost Bracket Sequence](https://codeforces.com/problemset/problem/3/D)

贪心。

先假设所有 `?` 都是 `)`，然后遍历括号串。
每次串不合法时将前缀 $a_i-b_i$ 最小的 `?` 改为 `(`，这部分可以用优先队列维护，若没有可更改的 `?` 则无可行解。
若最后串还是不合法则没有可行解，否则得到的一定是最优解。

先考虑有无可行解的正确性。将 `(` 看做 $1$，`)` 看做 $-1$，一个括号串合法当且仅当：

* 任意时刻串的前缀和 $\ge 0$；
* 最后串的前缀和 $=0$。

将 `)` 改为 `(` 的过程保证了第一个条件以及前缀和的实时最小。
也就是说，如果最后串还是不合法，那么前缀和 $>0$ 且无法更小，因此一定没有可行解。

然后考虑最小成本的正确性。

若我们将位置 $i$ 的 `)` 改为 `(`，则必然有位置 $j$ 的 `(` 变为 `)`，且原本与 $j$ 配对的 `)` 的位置 $k \ge i$。
这说明遍历到 $k$ 时，必然有将 $j$ 改为 `(` 的成本不高于将 $i$ 改为 `(` 的成本，因此这个变化下答案不优。

若我们将位置 $i$ 的 `(` 改为 `)`，则必然有位置 $j$ 的 `)` 变为 `(`，且此时与 $j$ 配对的 `(` 的位置 $k \ge i$。
这说明遍历到 $k$ 时，必然有将 $i$ 改为 `(` 的成本不高于将 $j$ 改为 `(` 的成本，因此这个变化下答案不优。

于是贪心策略正确，最终时间复杂度 $O(|S|\log|S|)$。
